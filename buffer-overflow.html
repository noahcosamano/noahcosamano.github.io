Buffer overflow exploit in C demonstrating how unsafe input handling can overwrite memory, redirect execution, and execute arbitrary code.

Overview

This project is an 8-page technical report analyzing a buffer overflow vulnerability in a C program. It explores how improper bounds checking allows attackers to manipulate stack memory and control program execution.

Key Concepts

  • Stack memory layout and function call structure
